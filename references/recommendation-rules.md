# Recommendation Reason Rules

Each recommendation reason must contain:

1. What problem the book solves.
2. Which stage it suits.
3. Why it is recommended, with evidence or context.

Template:

```markdown
[解决问题]
帮助理解 SAR 与药物结构优化。

[适合阶段]
相关课程学习中期或专业入门阶段。

[推荐原因]
与药物化学课程高度匹配，且为后续 CADD 学习打基础。
```

Avoid empty praise:

- 经典教材
- 值得阅读
- 大神推荐
- 内容丰富
- 很有帮助

These phrases are allowed only when followed by specific evidence, use case, or OPAC/external source context.
